# Шпаргалка по библиотекам (Developer Cheat Sheet)

> [!IMPORTANT]
> Проект поддерживает **Django 5.0+** и **Python 3.10+**.
> Поддержка Django 6.0 рассматривается как опциональная или будет включена позже при необходимости.

Этот файл содержит список рекомендованных библиотек для `codex_django`. Обработано на основе обсуждения.

## 1. Основа и API

*   ✅ **Django Ninja**: Супер штука! Применяем как основной инструмент для API.
*   ✅ **Pydantic**: Используем для валидации, особенно когда не работаем с классическими моделями Django.

## 2. Поиск и фильтрация (Advanced Search)

*   **django-filter**: Стандарт для фильтрации.
*   **django-watson**: Полнотекстовый поиск (без лишних зависимостей).
*   **django-haystack**: Если нужен Elasticsearch.
*   **django-elasticsearch-dsl**: Глубокая интеграция с ES.

## 3. Админка и UI

*   ✅ **Django Unfold**: Основной интерфейс админки.
*   ✅ **Django Import-Export**: Берем всё! Планируем проектировать по фильтрам таблиц из БД.
*   **Django Admin Sortable2**: Drag & Drop сортировка.

## 4. Качество кода и разработка

*   **django-extensions**: Shell_plus, show_urls и т.д.
*   **django-debug-toolbar**: Дебаг панель.
*   **django-stubs (mypy-django)**: Типизация для Django.
*   **Ruff**: Быстрый линтер.

## 5. База данных и модели

*   **django-simple-history**: Аудит изменений.
*   **django-lifecycle**: Удобная замена сигналам.
  * *Обновляемость*: Высокая (версия 1.2.7, март 2026).
  * *Стабильность (BC)*: Высокая. После версии 1.0.0 (2022г) критических поломок API не было. Основные изменения касаются поддержки новых версий Django/Python.
  * *Зависимость*: Средняя. Логика переносится в декораторы модели. Если уходить — придется возвращать в `save()` или сигналы. Делает код модели намного более читаемым (всё в одном месте).

## 6. Задачи и асинхронность

*   ✅ **ARQ**: Используем вместо Celery (на базе Redis).

## 7. Аутентификация

*   ✅ **django-allauth**: Комплексное решение для Auth. Планируем реализовать и использовать.
